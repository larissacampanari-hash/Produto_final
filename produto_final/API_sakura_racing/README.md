 Estrutura do Sistema

O sistema está organizado em três camadas principais:

 Banco de Dados

Responsável pelo armazenamento e gerenciamento das informações do sistema, incluindo tabelas, relacionamentos e consultas.

 Backend

Responsável pela lógica de negócio da aplicação, criação das rotas da API, tratamento das requisições e comunicação com o banco de dados.

 Frontend

Responsável pela interface do usuário, permitindo a interação com o sistema por meio de páginas web e integração com a API.

 Funcionalidades Implementadas

* Cadastro de informações no banco de dados.
* Consulta de registros através da API.
* Atualização de dados existentes.
* Exclusão de registros.
* Integração entre frontend e backend.
* Comunicação com banco de dados MySQL.
* Testes de rotas utilizando Postman.
